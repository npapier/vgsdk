Overview

vgSDK, a shortcut for Virtuals Graphical Software Development Kit, consists of a set of cross-platform C++ libraries which provides a complete framework for doing graphical related tasks.

Details

Here is a raw description of vgsdk features and structure.

Core Components

  • vgBase: standard typedefs, definitions and includes to deal with compiler or platform configurations issues
  • vgM: some basic linear algebra and geometry
  • vgd: the data repository stored in a scene graph
  • vgDebug a simple logging system

Algorithms

  • vgAlg: a collection of algorithms that could be applied on a node of the scene graph (compute topology from a triangle set)

Images

  • vgITK
  • OpenIL: image library encapsulated in vgd
  • Planned: vgCairo, vgOpenIL, vgImageMagick

Inputs & Outputs

  • vgTrian: .trian and .trian2 3D model loaders
  • 3DSMax: 7.x and 9.x importer/exporter
  • Planned: vgCollada

Engines

  • vge: evaluation of scene graph
  • vgeGL: evaluation/rendering of scene graph with OpenGL

User Interface

  • vgWX: wxWidgets specialization for vgsdk
  • Planned: gtk

Contributions

  • vgEgg: an incubator for external contributions

Examples

External Dependencies

  • gle: an OpenGL extension loader
  • glo: an OpenGL objects library
  • boost: free peer-reviewed portable C++ source libraries
  • ITK: an image library
  • OpenIL an image library
  • OpenGL: the cross-platform graphical API
  • wxWidgets: a cross-platform toolkits for user interfaces and more
  • The COLLADA Document Object Model (DOM): an API that provides a C++ object representation of a COLLADA XML instance document
